2. Implementazione del backend (Node + Express)
Ho creato un server http Express (server.js), collegato direttamente alle rotte grazie all'uso di app.use():
- si mette in ascolto sulla porta 3001
- riceve le richieste http (GET, POST, PUT, DELETE) efettuate tramite l'uso di Thunder Client
- applica le funzione middleware
- inoltra le richieste alle route giuste
- risponde con i dati JSON

6. completate le API REST necessarie
auth.js gestisce la rotta che registra/login utenti:
- legge i dati dal body
- controlla se l'email è già esistente
- crea un nuovo utente
- lo salva nel database
- genera un token JWT per tenerlo loggato
- risponde al client con token + dati utente
stessa cosa per i login
aggiunti GET, PUT e DELETE per i prodotti
cart.js gestisce la rotta del carrello di ogni utente:
- il cart è un array di oggetti (record, quantity)
- le API permettono di aggiungere/modificare e rimuovere i prodotti

LE API FANNO DA TRAMITE TRA FRONTEND E  DATABASE:

- risponde al frontend con dati in formato JSON
